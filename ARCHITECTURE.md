# 🏗️ Архитектура системы учета расходов

## Флоу работы

### 1. Управление категориями
**Страница:** `/categories`
- Создание/удаление категорий расходов
- Автоматическое добавление категории при создании копилки

### 2. Список месяцев
**Страница:** `/months`
- Отображение всех месяцев с суммами
- Кнопка создания нового месяца

### 3. Детали месяца
**Страница:** `/months/:id`

#### Функционал:

**A. Добавление плана (модалка)**
- Название дохода (Аванс, Зарплата)
- Дата дохода
- Сумма дохода
- Кнопка "Добавить расход в план"
  - Выбор категории (dropdown)
  - Плановая сумма
  - Кнопка уезжает вниз
  - Показывается "Не распределено: X ₽"

**B. Внесение фактического дохода (модалка)**
- Поле для ввода суммы
- Сумма **плюсуется** к existing actual_amount
- UPDATE income_items SET actual_amount = COALESCE(actual_amount, 0) + NEW_AMOUNT

**C. Внесение расхода (модалка)**
- Выбор категории (только из плановых)
- Поле для суммы
- Описание (опционально)
- Создает запись в expense_transactions
- Сумма **плюсуется** к итогу по категории

**D. Взять в долг (модалка)**
- Выбор копилки (dropdown)
- Поле для суммы
- Создает запись в income_debts
- Триггер автоматически:
  - Списывает с копилки
  - Добавляет к actual_amount дохода

**E. Итоговая таблица (read-only)**
Для каждого дохода отдельно:
```
Доход: Аванс (05.12.2025)
План: 120,000 ₽  |  Факт: 122,000 ₽

Категория       | План    | Факт    | Разница
----------------|---------|---------|----------
Еда             | 12,500  | 23,928  | -11,428
Бензин          | 5,000   | 5,097   | -97
...
----------------|---------|---------|----------
ИТОГО расходов  | 106,780 | 121,034 | -14,254
ОСТАТОК         | 13,220  | 966     |
```

**F. Итоги (внизу страницы)**
- Всего доход план/факт
- Всего расходы план/факт
- Общий остаток

## Модель данных

### income_items (доходы)
```sql
id, name, planned_amount, actual_amount (накопительная)
```

### expense_items (планы расходов)
```sql
id, income_item_id (в поле name), category_id, planned_amount
```

### expense_transactions (фактические траты)
```sql
id, income_item_id, category_id, amount, description
-- Фактическая сумма = SUM(amount) WHERE income_item_id = X AND category_id = Y
```

### income_debts (долги из копилок)
```sql
id, income_item_id, piggy_bank_id, amount
-- Триггер автоматически обновляет actual_amount дохода
```

## Адаптив

### Мобильная версия (<768px):
- Модалки на всю ширину
- Таблицы с горизонтальным скроллом
- Кнопки стакаются вертикально
- Уменьшенные отступы

### Планшеты (768px-1024px):
- Таблицы адаптируются
- 2 колонки для карточек

### Десктоп (>1024px):
- Полная ширина таблиц
- 3-4 колонки для карточек
